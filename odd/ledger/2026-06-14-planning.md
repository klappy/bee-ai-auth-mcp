Build minimal: auth core + pull egress only. Bide: inspected Bee's official MCP and rejected it as delivery (local-stdio only; never reaches mobile/web).

**[D] Credential security ladder.** Ship Tier 1 (self-host) + Tier 2 (hosted, hardened to git-auth parity). Tier 3 (naive paste-and-store) rejected. Tier 0 (Bee-issued OAuth / short-lived minted tokens) petitioned but blocked on Bee.

For the operator's personal need the wire is therefore *adopted*, not built. `bee-ai-auth-mcp` survives as the reusable substrate + interim Bee wire + MIT community artifact — not the operator's long-term personal wire.

**[D] Device recommendation: Omi for both operator (self-host = privacy) and friends (hosted MCP = zero setup).**

**[D] Build in parallel, four tracks.** A: adopt/verify Omi. B: Refinery (separate, device-agnostic). C: Tier-0 petition (field-wide). D: publish sanitized prior art.

**[D] Plan finalized into PRD v0.1**, written to the canonical PRD template. It is the single authoritative spec; the planning corpus are its inputs.

## Learnings

**[L] Custody is simpler than the ladder assumed — and already charted.** git-auth stores no secret (its GrantProps hold an `installationId` reference; it mints per call). bee-ai-auth-mcp can't mint, so it holds the Bee token — but inside `@cloudflare/workers-oauth-provider`'s already-encrypted per-grant props (decrypted to `ctx.props` per call; revocation = disconnect). No separate envelope store for Tier 1. This is exactly git-auth's own B1 roadmap pattern ("one encrypted refresh credential per grant inside the already-encrypted per-grant props"). KMS + Tier-0 gate are the Tier 2 hardening.

**[L] The ship/validation method is reusable and laptop-free.** Three-pass validation (requirements / four-surface reviewer sim / adversarial) with a fresh-context validator, confirmed at the wire ("client view is never evidence about the wire"; the connector layer caches tool schemas at connect time — reconnect to refresh). The manual OAuth dance (DCR → PKCE → operator authorizes in own browser → paste redirect → raw `initialize`/`tools/list`/`tools/call` → revoke) is a phone-only MCP-Inspector substitute. Adopt as Phase 1 DoD.
